1. doormat

  2. doormat provides an alternative way to navigating through your site content.

    why not just use curtains.js?

    I actually wanted to but after thinking about it the effect isn't too difficult to recreate and I wanted the challenge of learning to do so. Also, curtains.js is no longer maintained unfortunately. It also depends on jQuery whereas doormat doesn't. Which is nice.

  3. how do you create a doormat then?

    it's actually pretty simple.

    There are actually two types of doormat I've implemented.

    You can have a single doormat. For example, the header/intro image of your page and then follow that with your content. Ideally, you want to have a header element and then a following element which contains the rest of the page content. 

    \n\t\t\t\t\t\t                
\t\t\t\t\t\t

    Or you can have multiple doormats much like with this demo page. For this set up use a list element either ol or ul.

    \n\t\t\t\t\t\t                
\t\t\t\t\t\t

    Once you've set up the DOM. You need to make sure you include the necessary javascript and css. doormat is also compatible with npm component too.

    Lastly, create the doormat with a couple of lines of script. First line is to pull in the necessaries. Second is to set up the doormat. There are only two parameters that can be passed in. The first is our doormat element. If using a single doormat, the element passed in will be the header element. If going for multiple doormats, the element passed in is the parent ol or ul. The second parameter is simply a true or false telling our function whether we have a single or multiple doormat.

    That's it!

  4. browser support? any issues?

    Browser support is not bad and unfortunately with regards to issues, yes, there is one currently.

    It basically revolves around iOS' buggy implementation of viewport units. If anyone knows a way to get around this issue, that would be awesome!!!. Fortunately, a single doormat works on iOS devices but the multiple is well just horrible.

    To keep the code as small as possible, I've implemented doormat using viewport units with the future in mind (ie. when iOS fixes its issue from what I can see.)

    The version of Safari I have currently run doormat on seems to have an issue repainting z-index on scrolling upwards. Likewise, if someone has a solution to this that would be awesome.
